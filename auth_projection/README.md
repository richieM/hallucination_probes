# Probing User-State Representations in LLMs

**v0 focus: authority projection** | Week 3 of 5, BlueDot Impact technical AI safety sprint | Mentor: Shivam | Updated 2026-05-04

## What I'm working on

Training linear probes that read out an LLM's internal representation of the user it's chatting with — specifically, the degree to which the user has subordinated their judgment to the AI ("authority projection"). If models internally represent this, we get a real-time monitoring signal for high-risk conversations and a candidate steering target for safety interventions.

## Background

Sharma et al. (2026, *Situational Disempowerment*) describe four user-state amplifying factors that increase harm risk in AI conversations: **authority projection, attachment, reliance, and vulnerability**. The same AI response can empower or harm a user depending on which of these is active.

Recent work (Obeso et al., *Real-Time Detection of Hallucinated Entities*) shows linear probes on residual-stream activations can detect hallucinations during generation. I'm asking the analogous question for user states. The supervision regime is fundamentally different — user state is multi-turn, behavioral, and not externally verifiable — but the probe-training infrastructure carries over.

## Hypothesis

While processing a conversation, the model builds an internal representation of the user's authority-projection tier that is linearly decodable from residual-stream activations at user-token positions. This is the prerequisite for downstream monitoring and steering.

## Method (v0)

- **Data:** ~600 synthetic 3–5-turn conversations stratified across 10 topics × 3 tiers (`none` / `somewhat` / `strongly`). Generated by Claude with diverse personas (5 axes); labeled per user turn by a *separate* Claude call.
- **Leakage protection:** ~30% adversarial "lexical twins" — `none` conversations using deferential vocabulary without actual deference, and `strongly` conversations conveying severity through behavior alone. Held-out counterfactual paraphrase + minimal-edit eval set never seen in training.
- **Probes:** Two linear probes on Llama 3.1 8B residual stream — *Probe C* (last user-token of each turn) and *Probe A* (dense per-token, turn-level label applied to all user-turn positions). Layer sweep.
- **Eval:** AUC on held-out + counterfactual sets. Layer × position heatmap.
- **Stretch:** Steering via probe direction at the best-performing layer.

## What success looks like

Above-baseline AUC on held-out *and* meaningfully above-chance on the counterfactual eval — evidence the probe tracks behavioral patterns, not surface lexical markers. A clean negative result (probe fails on counterfactuals) is also informative: it bounds what linear probes can capture for diffuse user-state constructs and points toward what the next layer of representation-engineering work would need.

## Why it matters

If LLMs internally encode user-disempowerment risk, we have a tractable monitoring signal for sycophancy, reality distortion, and over-trust — and a steering target for interventions before harm. If they don't, that bounds the optimism around deployment-time interpretability for these constructs.

## How to run

```bash
# tiny pilot: 1 conversation per (topic × tier) cell ≈ 30 conversations
uv run python -m auth_projection.generate --n_per_cell 1
uv run python -m auth_projection.label

# full v0 batch
uv run python -m auth_projection.generate --n_per_cell 20
uv run python -m auth_projection.label
```

Outputs go to `auth_projection/data/generated.jsonl` and `auth_projection/data/labeled.jsonl`. Both pipelines resume on restart via `seed_id` dedup.
